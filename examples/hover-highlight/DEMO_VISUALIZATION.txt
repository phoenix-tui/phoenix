VISUAL KEY
───────────────────────────────────────────────────────────────────────────────

NORMAL BUTTON (not hovered):
  ╭────────────╮
  │  Button X  │  ← Single-line border (─ │ ╭ ╮ ╰ ╯)
  ╰────────────╯

HOVERED BUTTON (mouse over):
  ╔════════════╗
  ║  Button X  ║  ← Double-line border (═ ║ ╔ ╗ ╚ ╝)
  ╚════════════╝

STATUS INDICATORS:
- Hovered: (none)          → No button currently hovered
- Hovered: buttonX         → ButtonX is currently hovered
- Last clicked: (none)     → No button clicked yet
- Last clicked: buttonX    → ButtonX was last clicked (shown in green/bold)

═══════════════════════════════════════════════════════════════════════════════

INTERACTION FLOW DIAGRAM
───────────────────────────────────────────────────────────────────────────────

   Move Mouse → Hover Detection → Border Style Change → Status Update
                      ↓
              ProcessHover(pos, areas)
                      ↓
              EventHoverEnter/Leave
                      ↓
              Update hoveredID
                      ↓
              Re-render View()


   Click Mouse → Click Detection → Update lastClickedID → Status Update
                      ↓
              MouseActionRelease + hoveredID
                      ↓
              Store lastClickedID
                      ↓
              Re-render View()

TECHNICAL DETAILS
───────────────────────────────────────────────────────────────────────────────

Bounding Box System:
- Each button has a BoundingBox defining its hover area
- BoundingBox: (x, y, width, height) in terminal coordinates
- Mouse position checked against all boxes each motion event

Hover Detection:
- ProcessHover(position, areas) returns EventType
- EventHoverEnter → Mouse entered a component
- EventHoverLeave → Mouse left all components
- EventHoverMove → Mouse moved within same component
- EventMotion → Mouse moved outside all components

Visual Feedback:
- Normal: ╭─╮│╰─╯ (U+256D, U+2500, U+256E, U+2502, U+2570, U+256F)
- Hovered: ╔═╗║╚═╝ (U+2554, U+2550, U+2557, U+2551, U+255A, U+255D)
- Works in all terminals (no color dependency)

Performance:
- O(n) hover detection where n = number of buttons
- Efficient for typical UIs (even 100+ components)
- No allocations in hot path

TERMINAL COMPATIBILITY NOTES
───────────────────────────────────────────────────────────────────────────────

✓ WORKS PERFECTLY:
  - Windows Terminal
  - iTerm2 (macOS)
  - Alacritty
  - GNOME Terminal
  - Konsole
  - xterm
  - VS Code Terminal

⚠ REQUIRES SETUP:
  - SSH sessions (ensure mouse events forwarded)
  - tmux/screen (enable mouse mode: set -g mouse on)

✗ NOT SUPPORTED:
  - Very old terminals without mouse support
  - Text-only terminals (telnet, serial)
